# ラベル作成フロー

```
1. 顧客から発送先情報を受領
     ↓
2. 管理画面にログイン
     ↓
3. 必要情報を入力
     ↓
4. ラベル発行・印刷
     ↓
5. N&Uに共有
```

---

# 必須記載項目

## 基本項目（全件必須）

| 項目 | 内容 |
|------|------|
| 発送先氏名 | 顧客の氏名（ローマ字） |
| 発送先住所 | 国、州、市、番地、郵便番号 |
| 電話番号 | 顧客の連絡先 |
| **HSコード** | **9504.40** |

---

# HSコードについて

## トレーディングカード用

| 項目 | 値 |
|------|-----|
| HSコード | **9504.40** |
| 品目 | Playing cards |

## 記載例

| 項目 | 記載例 |
|------|--------|
| Description | Trading Cards (Pokemon/Yu-Gi-Oh/etc.) |
| HS Code | 9504.40 |
| Quantity | 10 pcs |
| Value | $500 |

# 顧客希望があれば追加

| 項目 | 備考 |
|------|------|
| 州納税者番号 | 米国の一部州で必要 |
| インボイス | 商業インボイスの添付 |
| FedEx番号 | 顧客の法人アカウント番号 |

商業目的なので → **NG**

# 価格申告について

## ルール

- **必ず実際の取引価格を正確に申告**
- 顧客から「安く書いて」と言われても断る

## 断り方

> 「税関リスクがあるため対応できません」

---

# よくあるトラブル

| トラブル | 対応 |
|----------|------|
| 住所不備で返送 | 顧客に正確な住所を再確認 |
| 税関で止まった | 顧客に追加書類の提出を依頼 |
| 紛失・破損 | 追跡番号で調査、保険対応 |

> 配送トラブルは早めに報告
