## 3. ラベル作成手順

### ■ 使用システム

- **DHL**：DHL Express管理画面
- **FedEx**：FedEx Ship Manager

### ■ 作成フロー

```
1. 顧客から発送先情報を受領
2. 管理画面にログイン
3. 必要情報を入力
4. ラベル発行・印刷
5. N&Uに共有
```

---

## 4. 必須記載項目

### ■ 基本項目（全件必須）

| 項目 | 内容 |
|------|------|
| **発送先氏名** | 顧客の氏名（ローマ字） |
| **発送先住所** | 国、州、市、番地、郵便番号 |
| **電話番号** | 顧客の連絡先 |
| **HSコード** | **9504.40**（トレーディングカード用） |

### ■ 顧客希望があれば追加

| 項目 | 備考 |
|------|------|
| 州納税者番号 | 米国の一部州で必要 |
| インボイス | 商業インボイスの添付 |
| FedEx番号 | 顧客の法人アカウント番号 |

## 6. 国別ルール（重要）

### ■ オーストラリア 🇦🇺

| 項目 | ルール |
|------|--------|
| 推奨業者 | **DHL** |
| 金額上限 | **1梱包 1,000 AUD以下** |
| 理由 | 1,000AUD以下は関税免除 |

```
・必ず1,000AUD以下になるよう分割
・顧客にとって関税がかからずお得
・複数梱包になる場合は事前に顧客に確認
```

---

### ■ シンガポール 🇸🇬

| 項目 | ルール |
|------|--------|
| 推奨業者 | DHL / FedEx |
| 金額上限 | **希望があれば 400 SGD以下** |
| 理由 | 400SGD以下は関税免除 |

```
・顧客から希望があれば分割対応
・利益と工数を考慮して判断
・希望がなければ通常発送でOK
```

---

### ■ アメリカ 🇺🇸

| 項目 | ルール |
|------|--------|
| 推奨業者 | **FedEx** |
| 金額上限 | **1梱包 2,500 USD以下** |
| 理由 | 2,500USD超は通関で止まるリスク高 |

```
・2,500USD超の場合は分割必須
・顧客に事前確認「分割しますか？」
```

### ■ アメリカの関税対応（重要）

| 項目 | 内容 |
|------|------|
| 関税率 | **約16%** |
| 対応方法 | **事前徴収 + DDP設定** |

**手順：**

```
1. 見積もり時に関税16%を上乗せして請求
2. FedExラベル作成時に「関税支払い」を設定
   └ 「受取人払い」→「発送人払い（マイアカウント）」に変更
3. こちらで関税を支払うDDP（Delivered Duty Paid）形式
```

**メリット：**
- 顧客が通関時に支払う手間なし
- 受取拒否リスクが減る
- 顧客満足度UP

## 8. 注意事項

### ■ 絶対NGな対応

| NG | 理由 |
|----|------|
| **アンダーバリュー**（実際より低い価格申告） | 違法、税関リスク |
| **ギフト扱いでの送付** | 商業目的なのでNG |

### ■ 価格申告について

```
・必ず実際の取引価格を正確に申告
・顧客から「安く書いて」と言われても断る
・理由：「税関リスクがあるため対応できません」と伝える
```

---

## 9. トラブル対応

### ■ よくあるトラブル

| トラブル | 対応 |
|----------|------|
| 住所不備で返送 | 顧客に正確な住所を再確認 |
| 税関で止まった | 顧客に追加書類の提出を依頼 |
| 紛失・破損 | 追跡番号で調査、保険対応 |

### ■ エスカレーション

```
・配送トラブルは早めに報告
・保険対応が必要な場合は山崎に相談
```
