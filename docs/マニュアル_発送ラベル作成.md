## 2. 使用する配送業者

| 業者 | 使用率 | 特徴 |
|------|--------|------|
| **DHL** | 約50% | 速い、追跡充実 |
| **FedEx** | 約50% | 速い、追跡充実 |
| その他 | 1%以下 | 特殊な場合のみ |

---

## 3. ラベル作成手順

### ■ 使用システム

- **DHL**：DHL Express管理画面
- **FedEx**：FedEx Ship Manager

### ■ 作成フロー

```
1. 顧客から発送先情報を受領
2. 管理画面にログイン
3. 必要情報を入力
4. ラベル発行・印刷
5. N&Uに共有
```

---

## 4. 必須記載項目

### ■ 基本項目（全件必須）

| 項目 | 内容 |
|------|------|
| **発送先氏名** | 顧客の氏名（ローマ字） |
| **発送先住所** | 国、州、市、番地、郵便番号 |
| **電話番号** | 顧客の連絡先 |
| **HSコード** | **9504.40**（トレーディングカード用） |

### ■ 顧客希望があれば追加

| 項目 | 備考 |
|------|------|
| 州納税者番号 | 米国の一部州で必要 |
| インボイス | 商業インボイスの添付 |
| FedEx番号 | 顧客の法人アカウント番号 |

---

## 5. HSコードについて

### ■ トレーディングカード用コード

```
HSコード：9504.40
品目：Playing cards（トランプ・トレーディングカード）
```

### ■ 記載例

| 項目 | 記載例 |
|------|--------|
| Description | Trading Cards (Pokemon/Yu-Gi-Oh/etc.) |
| HS Code | 9504.40 |
| Quantity | 10 pcs |
| Value | $500 |

## 6. 注意事項

### ■ 絶対NGな対応

| NG | 理由 |
|----|------|
| **アンダーバリュー**（実際より低い価格申告） | 違法、税関リスク |
| **ギフト扱いでの送付** | 商業目的なのでNG |

### ■ 価格申告について

```
・必ず実際の取引価格を正確に申告
・顧客から「安く書いて」と言われても断る
・理由：「税関リスクがあるため対応できません」と伝える
```

---

## 7. トラブル対応

### ■ よくあるトラブル

| トラブル | 対応 |
|----------|------|
| 住所不備で返送 | 顧客に正確な住所を再確認 |
| 税関で止まった | 顧客に追加書類の提出を依頼 |
| 紛失・破損 | 追跡番号で調査、保険対応 |

### ■ エスカレーション

```
・配送トラブルは早めに報告
・保険対応が必要な場合は山崎に相談
```

## 8. チェックリスト

ラベル作成時に確認：

- [ ] 発送先氏名が正確
- [ ] 住所が完全（国、州、市、番地、郵便番号）
- [ ] 電話番号が記載されている
- [ ] HSコード（9504.40）が入力されている
- [ ] 価格が実際の取引価格と一致
- [ ] 顧客希望の追加情報があれば記載
